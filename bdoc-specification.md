# Спецификация открытого пакетного формата BDoc (v0.1-composite)

Спецификация описывает архитектуру открытого формата **BDoc (`.bdoc`)**, ориентированного на верстку, реставрацию, очистку и конвертацию сложных многостраничных документов (книг, журналов, газет). Формат разделяет смысл (семантику), геометрию размещения (layout) и бинарные ресурсы.

## 1. Физическая структура (Контейнер)

Файл `.bdoc` является стандартным **ZIP-архивом** без сжатия или с минимальным сжатием (для высокой скорости чтения байтов). Структура каталогов внутри архива:

```
my_book.bdoc (ZIP-архив)
 ├── manifest.json       # Метаданные книги и карта документов
 ├── styles.json         # Стили текста и графических объектов
 ├── stories/
 │    ├── story-1.json   # Логический текстовый поток №1 (Глава 1)
 │    └── story-2.json   # Логический текстовый поток №2 (Глава 2)
 ├── pages/
 │    ├── page-1.cbor    # Бинарная геометрия страницы 1 (быстрый парсинг)
 │    ├── page-2.cbor    # Бинарная геометрия страницы 2
 │    └── ...
 └── resources/          # Бинарные ассеты
      ├── page-1-scan.jpg # Растровый скан-подложка для реставрации оригинала
      └── font-1.ttf      # Встроенный шрифт
```

------

## 2. Логическая структура (Спецификация файлов)

### 2.1. `manifest.json`

Корень документа, точка входа для редактора. Отвечает за идентичность и предоставляет «карту» для ленивой загрузки страниц.

json

```
{
  "id": "doc-uuid-12345",
  "title": "Демонстрационный документ BDoc",
  "documentType": "book",
  "version": "0.1-composite",
  "language": "ru-RU",
  "pages": [
    { "index": 1, "id": "page-1", "file": "pages/page-1.cbor" },
    { "index": 2, "id": "page-2", "file": "pages/page-2.cbor" }
  ],
  "stories": [
    { "id": "story-1", "file": "stories/story-1.json" }
  ]
}
```

Используйте код с осторожностью.



### 2.2. `stories/story-X.json`

Хранит логический поток текста (семантический слой) полностью изолированно от геометрии страниц (preliminar... pp. 4-5). Это позволяет очищать текст от рекламы и мусора без риска сломать макет (preliminar... pp. 1, 5).

json

```json
{
  "id": "story-1",
  "paragraphs": [
    { "role": "heading", "styleRef": "h1", "text": "Глава первая" },
    { "role": "body", "styleRef": "main-text", "text": "Текст очищен от рекламы." }
  ]
}
```





### 2.3. `pages/page-X.cbor` (Спецификация в JSON-виде)

Описывает жесткую геометрию конкретной страницы (слои, фреймы, координаты объектов) (preliminar... pp. 3-4). Для максимальной скорости работы на 1000+ страницах этот файл сохраняется в бинарном формате **CBOR** (парсится мгновенно без чтения текстовых тегов).

json

```json
{
  "id": "page-1",
  "index": 1,
  "width": 595.0,
  "height": 842.0,
  "unit": "mm",
  "layers": [
    { "id": "layer-bg", "name": "Сканы-подложки", "role": "background", "visible": true, "opacity": 0.4 },
    { "id": "layer-text", "name": "Основной текст", "role": "text", "visible": true, "opacity": 1.0 }
  ],
  "objects": [
    {
      "type": "ImageFrame",
      "id": "scan-1",
      "layerRef": "layer-bg",
      "geometry": { "x": 0.0, "y": 0.0, "width": 595.0, "height": 842.0 },
      "assetRef": "resources/page-1-scan.jpg"
    },
    {
      "type": "TextFrame",
      "id": "frame-1",
      "layerRef": "layer-text",
      "storyRef": "story-1",
      "geometry": { "x": 70.0, "y": 120.0, "width": 455.0, "height": 600.0 }
    }
  ]
}
```

# Пошаговый план миграции проекта `bdoc-editor`

Чтобы перевести вашу JavaFX-программу со старой монолитной XML-схемы на новую пакетную спецификацию, выполните следующие шаги:

## Шаг 1: Замена JAXB на Jackson (JSON + CBOR)

Убрать зависимости JAXB из проекта. Добавить в  `build.gradle` библиотеки **Jackson**:

- `jackson-databind` (для работы с JSON файлами манифеста и историй)
- `jackson-dataformat-cbor` (для бинарной упаковки страниц)

## Шаг 2: Создание класса `BdocContainer` (Работа с ZIP)

Написать утилитарный класс для чтения/записи ZIP-архивов на лету без распаковки на диск с помощью встроенного в Java API (`java.nio.file.FileSystems`):

java

```java
Map<String, String> env = new HashMap<>();
env.put("create", "true");
URI uri = URI.create("jar:file:" + bdocFile.toURI().getPath());
try (FileSystem zipfs = FileSystems.newFileSystem(uri, env)) {
    // Теперь zipfs позволяет работать с архивом как с обычной папкой
    Path manifestPath = zipfs.getPath("/manifest.json");
    // Записываем или читаем через Jackson
}
```



## Шаг 3: Реализация «Ленивой загрузки» (Lazy Loading) в Модели

Переписать`BdocXmlSerializer` (переименовав его, например, в `BdocContainerSerializer`):

1. При вызове метода `load(File file)` программа должна прочитать только `manifest.json` и файлы из папки `stories/`.
2. Объект `DocumentModel` инициализируется, но его поле `List<PageModel> pages` остается пустым (или заполняется прокси-объектами). Наполняется только дерево `ListView` .
3. Добавьте метод `PageModel loadPage(int index)`, который будет обращаться к ZIP-контейнеру, вытаскивать байты конкретного `pages/page-X.cbor`, десериализовать его через Jackson CBOR-парсер и возвращать для отрисовки.

## Шаг 4: Модернизация `PageRenderer` для поддержки сканов

В классе `PageRenderer` расширить метод `renderObject` 

1. Добавить поддержку нового объекта `ImageFrame` 
2. Метод должен брать путь `assetRef` (например, `resources/page-1-scan.jpg`), запрашивать поток байт этой картинки из открытого ZIP-архива, преобразовывать в `javafx.scene.image.Image` и рисовать на Canvas через `gc.drawImage()` с учетом прозрачности (`opacity`), заданной в настройках слоя.

## Шаг 5: Покрытие тестами новой структуры

Переписать `BdocXmlSerializerTest` под работу с ZIP-контейнером. Убедитесь, что тест проверяет "Roundtrip" (сохранение книги на 1000 условных страниц, ленивое извлечение случайной 500-й страницы и проверка равенства координат её фреймов)
